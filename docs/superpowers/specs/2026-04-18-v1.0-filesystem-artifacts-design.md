# v1.0 Design Spec — Filesystem Artifact Generation

> Status: Draft
> Author: d3vn0mi + Claude
> Date: 2026-04-18

---

## 1. Problem Statement

Real incidents aren't just event logs. Analysts examine Prefetch files,
Amcache entries, and $MFT records to corroborate findings from event data.
ArtiForge generates event logs but no forensic filesystem artifacts, so
trainees can't practice cross-artifact correlation — the skill of verifying
that multiple evidence sources tell the same story.

## 2. Solution

Auto-generate Prefetch (binary `.pf`), Amcache (JSON), and $MFT (JSON)
artifacts from process execution events in the generated bundle. Opt-in via
`forensic_artifacts: true` on the attack spec. Only attack events
(phase_id != 0) trigger artifact generation.

## 3. Design Goals

- Opt-in, default off — existing labs unaffected
- Auto-derived from event data — no additional YAML authoring needed
- Prefetch in real binary format (parseable by PECmd)
- Amcache and $MFT as structured JSON (same fields as tool output)
- Cross-artifact correlation: all three reference the same executables
  with consistent timestamps and hashes

## 4. YAML Integration

### 4.1 Opt-In Flag

```yaml
attack:
  base_time: "2026-02-19T09:12:00Z"
  forensic_artifacts: true          # NEW — default: false
  phases:
    - id: 1
      ...
```

When `true`, the engine scans the generated bundle after all phases run
and produces filesystem artifacts for every unique process execution
(Sysmon 1 and Security 4688) in attack events.

### 4.2 Model Change

```python
class AttackSpec(BaseModel):
    base_time: datetime
    malicious_account: str = "svc_backup_admin"
    phases: list[Phase] = Field(default_factory=list)
    noise: list[NoiseSpec] = Field(default_factory=list)
    forensic_artifacts: bool = False     # NEW
```

## 5. Artifact Types

### 5.1 Prefetch (Binary `.pf`)

One `.pf` file per unique executable path per host.

**Format:** Windows 10+ Prefetch version 30 (uncompressed). Layout:

| Offset | Size | Description |
|--------|------|-------------|
| 0 | 4 | Version: 30 (0x1E000000) |
| 4 | 4 | Magic: `MAM\x04` (0x4D414D04) |
| 8 | 4 | Unknown (0x11) |
| 12 | 4 | File size |
| 16 | 60*2 | Executable name (UTF-16LE, padded to 60 chars) |
| 136 | 4 | Prefetch hash |
| 140 | 4 | Unknown (0) |
| 144 | 8 | Last run time (FILETIME) |
| 152 | 24 | Previous run times (3 x FILETIME, zeroed if run_count < 4) |
| 176 | 4 | Run count |
| 180 | 4 | Unknown (0) |

Total header: 184 bytes. File may include a references section (list of
loaded DLLs/files) but this is optional for training purposes.

**Prefetch hash algorithm:** Standard Windows Prefetch path hash — SSDEEP of
the uppercase full path. For simplicity, we use a CRC32-based approximation
that produces consistent 8-hex-char hashes.

**Filename:** `EXECUTABLE.EXE-AABBCCDD.pf` where `AABBCCDD` is the
prefetch hash of the full path.

**Data source:** Sysmon 1 events — `Image` field provides the exe path,
`UtcTime` provides the timestamp, `Hashes` provides SHA256/MD5 if present.

### 5.2 Amcache (`amcache_entries.json`)

One JSON file per host. Array of entries:

```json
[
  {
    "full_path": "C:\\Temp\\mimikatz.exe",
    "sha1": "DA39A3EE5E6B4B0D3255BFEF95601890AFD80709",
    "file_size": 1250816,
    "first_run": "2026-02-19T09:15:05Z",
    "last_modified": "2026-02-19T09:14:00Z",
    "publisher": "",
    "file_version": "10.0.19041.1",
    "pe_header_checksum": "0x00000000",
    "original_filename": "mimikatz.exe"
  }
]
```

**Data source:** Sysmon 1 events — `Image` (full_path), `Hashes`
(SHA1 extracted or generated), `FileVersion`, `OriginalFileName`,
`Company` (publisher). Timestamp from event.

### 5.3 $MFT (`mft_entries.json`)

One JSON file per host. Array of records:

```json
[
  {
    "record_number": 131073,
    "filename": "mimikatz.exe",
    "parent_directory": "C:\\Temp",
    "file_size": 1250816,
    "created": "2026-02-19T09:14:00Z",
    "modified": "2026-02-19T09:14:00Z",
    "accessed": "2026-02-19T09:15:05Z",
    "entry_modified": "2026-02-19T09:14:00Z",
    "is_directory": false,
    "in_use": true
  }
]
```

**Data source:** Sysmon 1 events — `Image` parsed into filename +
parent_directory. Timestamps derived from event timestamp (created =
a few seconds before execution, accessed = execution time). Sysmon 11
(FileCreate) events can provide creation timestamps if available.

## 6. Cross-Artifact Correlation

For each unique executable in attack events, all three artifact types
reference consistent data:

| Field | Prefetch | Amcache | $MFT |
|-------|----------|---------|------|
| Executable name | Header field | `full_path` | `filename` |
| Execution time | `last_run_time` | `first_run` | `accessed` |
| Hash | — | `sha1` | — |
| File version | — | `file_version` | — |
| Directory | — | dirname of `full_path` | `parent_directory` |

The orchestrator collects all process execution events, deduplicates by
(host, image_path), and passes each unique process to all three generators
with the same metadata.

## 7. Architecture

### 7.1 File Structure

```
artiforge/generators/
├── forensic_artifacts.py    # Orchestrator: scans bundle, delegates
├── prefetch.py              # Prefetch binary generator
├── amcache.py               # Amcache JSON generator
├── mft.py                   # $MFT JSON generator
```

### 7.2 Orchestrator

```python
# artiforge/generators/forensic_artifacts.py

def generate(bundle: ArtifactBundle, output_dir: Path) -> list[Path]:
    """Generate forensic artifacts from attack process execution events.

    Scans for Sysmon 1 and Security 4688 events, deduplicates by
    (host, image_path), and produces Prefetch + Amcache + $MFT.
    """
```

Called from `engine.py` (or `cli.py`) after bundle generation, only when
`spec.attack.forensic_artifacts is True`.

### 7.3 ProcessInfo Dataclass

Shared between all three generators:

```python
@dataclass
class ProcessInfo:
    image_path: str              # C:\Temp\mimikatz.exe
    image_name: str              # mimikatz.exe
    parent_dir: str              # C:\Temp
    first_run: datetime          # earliest execution timestamp
    run_count: int               # number of times executed
    hashes: dict[str, str]       # {"SHA256": "...", "MD5": "..."}
    file_version: str            # from Sysmon 1 FileVersion field
    original_filename: str       # from Sysmon 1 OriginalFileName
    company: str                 # from Sysmon 1 Company field
    host: str                    # hostname
```

## 8. Engine/CLI Integration

### 8.1 Engine

After generating all events and noise in `engine.run()`, the bundle is
returned as before. The forensic artifact generation happens in the CLI
`generate` command, not the engine — this keeps the engine focused on
event generation.

### 8.2 CLI

In the `generate` command, after writing events and file artifacts:

```python
if spec.attack.forensic_artifacts:
    from artiforge.generators import forensic_artifacts
    forensics_dir = run_dir / "forensics"
    forensic_files = forensic_artifacts.generate(bundle, forensics_dir)
    written.extend(forensic_files)
    click.echo(f"  [forensics] → {forensics_dir}  ({len(forensic_files)} artifacts)")
```

### 8.3 Output Structure

## 10. Testing Strategy

- **Prefetch tests:** Binary header magic, version, executable name encoding,
  hash computation, run count, timestamp, file size matches header field
- **Amcache tests:** JSON structure, required fields present, SHA1 hash
  format, timestamps match event data
- **$MFT tests:** JSON structure, filename/parent_dir parsed correctly,
  4 NTFS timestamps present, record numbers unique
- **Orchestrator tests:** Deduplication by (host, image_path), run_count
  aggregation, only attack events processed, noise events skipped
- **Cross-correlation test:** Generate UC3 with `forensic_artifacts: true`,
  verify every Prefetch exe has a matching Amcache entry and $MFT record
- **Opt-out test:** UC3 without `forensic_artifacts` produces no forensics dir
- **Backward compat:** Existing labs unchanged

2. Prefetch binary generator
3. Amcache JSON generator
4. $MFT JSON generator

## 12. Out of Scope

- Real Windows registry hive (regf) format for Amcache
- Real NTFS $MFT binary format
- ShimCache / AppCompatCache
- Browser history artifacts
- Windows Event Log `.evt` (legacy) format
- Per-phase or per-event artifact suppression/override
- Linux filesystem artifacts
